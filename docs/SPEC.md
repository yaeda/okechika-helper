# 仕様 (SPEC)

## 1. 概要

- 本プロジェクトは、ゲーム「桶地下」を遊ぶためのヘルパーツール群を提供する
- 現時点の対象プロダクトはブラウザ拡張とする
- ブラウザ拡張の目的は、桶地下文字の解読作業を支援し、読み取りと学習を効率化すること

## 2. 対象プロダクト: ブラウザ拡張

### 2.1 採用技術

- WXT
- React
- Tailwind CSS

### 2.2 提供機能

1. ルビ表示機能
- 登録済みの解読テーブルを参照し、対象ページ上の桶地下文字にルビを自動表示する
- 未登録の文字は `?` を表示する
- 空白文字（スペース・改行等）と句読点・引用符などの記号は翻訳対象外とし、ルビを表示しない
- 対象はユーザーが管理画面で設定したルート URL リストとする
- 初期対象 URL は以下とする
  - `https://www.pub-riddle.com/` 配下
  - `https://www.qtes9gu0k.xyz/` 配下
- ルート URL 判定では `www` の有無を同一扱いとする（例: `https://www.qtes9gu0k.xyz/` と `https://qtes9gu0k.xyz/`）
- `srcdoc` / `about:blank` フレームでは `document.referrer` の URL を判定に利用する

2. 解読テーブル更新機能
- ユーザーがテキストを選択した時点で tooltip を表示する
- tooltip 上で対応文字列を入力すると、解読テーブルを更新できる
- 既存エントリに別の文字列を入力した場合は上書きする
- tooltip 上で `?` を入力した文字は「未登録に戻す」操作として扱い、既存エントリを削除する
- 空白文字（スペース・改行等）と句読点・引用符などの記号は更新対象文字数に含めない
- tooltip 上に、選択文字列をそのままコピーするボタンを提供する
- tooltip 上に、選択文字列を解読テーブルでデコードした文字列（未登録は `?`）をコピーするボタンを提供する
- tooltip 上に、選択文字列を `?s=` クエリとして現在マッチ中の対象ルート URL に付与して検索ページへ遷移するボタンを提供する

## 3. データ

- 解読テーブルを保持し、ルビ表示機能と更新機能の両方で利用する
- 保存先は拡張機能の `chrome.storage.sync` を利用する
- テーブル全体の最終更新日時（`updatedAt`）を 1 つ保存する
- `updatedAt` は ISO 8601 形式の UTC 文字列で保持する
  - 例: `2026-03-04T10:00:00.000Z`
- 初期テーブルは空で開始する

## 4. 文字判定ルール

- `font-family` が `UnknownGlyphs` または `Underground` の文字列を「桶地下文字」と判定する

## 5. 管理機能

- 変換テーブルを閲覧できる管理ページを提供する
- 変換テーブルは CSV 形式でエクスポートできる
- CSV エクスポート時のファイル名は `okechika-table-<翻訳済み件数>-<hash>.csv` 形式とする
- 変換テーブルは CSV 形式でインポートできる
- CSV の列は `source,target` とする（`updatedAt` はテーブル全体のメタ情報として別管理）
- CSV インポート時は既存テーブル全体を置き換える
- CSV インポート時に `target` が `?` の行は削除扱いとし、保存しない
- `target` が `?` のエントリは保存しないため、CSV エクスポートにも含めない
- 管理ページ上の定義済み文字テーブルは「文字」と「数字」の 2 つに分割表示する
- 管理ページの変換テーブルで「変換前」に専用フォント（`UnknownGlyphs`）を適用できる
- 専用フォント適用は管理ページ上のスイッチで ON/OFF を切り替えられる
- 管理ページ上部ヘッダーと各パネルは、変換テーブル幅基準の同一カラムに揃えてページ中央に配置する
- 管理ページの変換テーブルで「両方表示」を選択した場合、各セルは `変換前 > 変換後` の形式で表示する
- 管理ページの変換テーブルの「変換後」は、セルをダブルクリックして直接編集できる
- 変換前表示 / 変換後表示 / 両方表示のいずれでも、ダブルクリックしたセル全体を入力ボックスに置き換えて編集する
- セル編集はフォーカスアウトまたは Enter で適用し、Esc でキャンセルできる
- セル編集で `?` または空文字を適用した場合は削除扱いとする
- 管理ページの変換テーブルで未解析（`?`）が含まれる表示は、対象文字と区切りをグレー表示にする
- 対象ルート URL を追加・削除できる
- 対象ルート URL 一覧の各行から、登録済み URL を新規タブで開ける
- 対象ルート URL 一覧は初期表示で伏せ字とし、表示/非表示を切り替えるボタンを提供する
- 初期対象 URL に戻す「Reset to Default」ボタンを提供する
- 初期対象 URL 以外を追加する際は、対象サイトへのアクセス権限を要求し、許可された URL のみを有効化する
- 拡張機能アイコンをクリックした際に管理ページ（Options）を開く
- 管理ページ上部ラベル「桶地下 helper」の右側に、拡張機能バージョン（`v{version}`）を表示する
- 管理ページ上部ヘッダーの右側に、不具合報告リンクと更新内容リンクを表示する
- 管理ページ上部の説明テキストは、タイトルのすぐ右に並べて表示する
- 狭い画面でテーブル幅が広い場合は、ページ全体ではなくテーブル領域のみ横スクロールする
- 横スクロール可能なテーブルでは、左右端にフェードを表示して続きがあることを示す
- 管理ページに「相互変換」を表示する
  - 「桶地下→日本語」「日本語→桶地下」の 2 タブで表示を切り替える
  - 桶地下文字から日本語への変換入力・結果表示・コピーを提供する
  - 日本語から桶地下文字への変換入力・候補選択・結果表示・コピーを提供する
  - 日本語から桶地下文字への変換では、同一文字列に複数候補がある場合に候補を選択できる
- 管理ページに「権利について」を表示する
  - 桶地下が第四境界のコンテンツである旨
  - 第四境界トップページへのリンク
  - 「桶地下 調査の手引き」へのリンク
  - 本拡張機能がファンメイド作品であり、第四境界とは無関係で、権利侵害の意図がない旨
  - この拡張機能へのコンタクト先（X / GitHub）へのリンク

## 6. 入力仕様

- 入力文字種は制限しない（ひらがな・カタカナ・漢字・アルファベット・数字・記号を許可）
- 入力可能な最大文字数は「選択した桶地下文字の数」と同じとする
- ただし、数字テーブルの桶地下文字を 1 文字だけ選択している場合は、2 桁以上の数字文字列を 1 つの変換結果として入力できる

## 7. 将来拡張

- 他ユーザー作成の解析テーブルをインポート可能にする
- 複数テーブルから利用するテーブルを選択可能にする

## 8. 開発時の注意

- `iframe`（特に `srcdoc` / `about:blank`）内での動作確認は、`build` 版の拡張を推奨する
- lint 実行時は生成物ディレクトリ（`.output/`, `.wxt/`）を対象外とする
